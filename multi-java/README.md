#### 基础

```text
并发：多个任务在单核CPU竞争资源；
并行：多个任务在多核CPU环境下，并行执行。
```

#### 进行的并发与多线程的并发

```text
1、进程间的通讯比较复杂，而线程间的通信比较简单，通常情况下，我们需要使用共享资源，这些资源的在线程间的通讯比较容易；
（后续在业务量逐渐庞大的大型时，进程间的并发也是一种解决方案，也就是我们所说的分布式系统，此处讲的是线程，所以针对的都是小型系统）
2、进程的重量级的，而线程的轻量级的，故多线程方式的系统开销更小；
```

#### 进程与线程的区别
````text
进程是一个独立的运行环境，而线程是进程中执行的一个任务，他们的本质区别是，是否单独战友内存地址空间与其他系统资源：
    1、进程单例占有一定的内存地址空间，所以进程间存在内存隔离，数据是分开的，数据共享复杂但是同步简单，各个进程之间互不各个京城之间互不干扰，
        而进程中的现场则共享所属进程占有的内存地址空间和资源，数据共享，但同步相对复杂；
        
    2、进程单例占有一定的内存地址空间，一个进程出现问题不会影响其他进程，可靠性高，一个线程崩溃，可能影响整个程序的稳定性，可靠性较低，这也就
        要求我们在编码的时候，尽量提高多线程程序的可靠性；
        
    3、进程单独占有一定的内存地址空间，进程的创建和销毁不仅需要保存寄存器和栈信息，还需要资源分配回收以及页调度，开销较大，而现场只需要保存寄存器和栈信息，开销较小。
    
    4、另一个重要的区别是，进程的操作性进行资源分配的基本单位，而线程是操作系统进行调度的基本单位（CPU分配时间的单位）。
````
